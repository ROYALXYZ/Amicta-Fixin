# Context Archive
> Historical decisions and notes. Reference only.

## Archived: 2026-07-21
### Architecture Decisions
- Produk memakai SaaS B2B multi-tenant terpusat: satu deployment dan PostgreSQL bersama melayani banyak organisasi.
- Semua data operasional wajib memiliki `organization_id`; Laravel menentukan tenant context dari pengguna terautentikasi dan membatasi query per organisasi.
- Platform Owner membuat organisasi serta Admin Organisasi pertama; admin mengelola operasional tenantnya.
- Tenant context ditentukan dari subdomain organisasi (mis. `amikom.ranggapasha.my.id`), lalu Laravel memvalidasinya terhadap organisasi akun login; browser tidak mengirim `organization_id` tepercaya.
- Penghuni dapat melapor untuk semua unit aktif dalam organisasinya. Admin dapat membatalkan tiket belum selesai dengan alasan; `DIBATALKAN` bersifat final.
- Foto wajib bertipe JPEG/WebP maksimal 2 MB, disimpan privat di Supabase Storage dan disajikan dengan signed URL setelah otorisasi.
- Base domain tenant dikonfigurasi melalui `TENANCY_BASE_DOMAIN`; perubahan domain hanya memerlukan environment configuration dan DNS wildcard, tanpa perubahan data tenant.
